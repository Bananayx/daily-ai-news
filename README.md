# daily-ai-news



## 一个由AI撰写的每日AI晚报。



在 TRAE Work 的 自动化 - Code模式 使用以下提示词实现，每日17:00触发：



```text

搜索今日 AI 行业的热点新闻，覆盖以下方面：

1. 重要产品发布或功能更新（如 OpenAI、Google、Anthropic 等公司动态）

2. 融资事件与行业并购

3. 技术突破或重要论文发布

4. 行业标准与规范动态（如 AI 安全框架、数据治理标准的进展）

输出要求：

- 按重要性排序

- 每条新闻包含：标题、详情、信息来源

- 聚焦技术进展和商业动态层面

- 最后附一段 2-3 句的“今日趋势点评”

- 新闻必须具有强时效性，必须是今日的，昨天、前天、以前的一概不要

将index.html作为AI晚报的首页，设计规范参照`spec-homepage.md`，其中顶部导航栏左侧文字写“每日AI晚报”，title改为“每日AI晚报”，关于网站写README.md中的内容，关于作者内容写“懒得写”

将yy-mm-dd.html作为每日AI新闻的内容页，设计规范参照`spec-daily-news.md`，其中顶部导航栏左侧文字写“每日AI晚报”，title改为“yyyy.mm.dd - 每日AI晚报”，页面中每条新闻都要附带与其对应的信息来源的跳转链接，每条新闻不必显示具体时刻（hh:mm）

注意index.html历史新闻部分，你要先查看已有哪些yy-mm-dd.html，时间线上只显示已有的

每次你的任务触发时，都要更新index.html到最新状态，并新增最新的yy-mm-dd.html，旧的不要删。两个页面的页脚的年份都要更新到现在的年份。并且直接push到origin/main而不要提交PR

```



设计规范（`spec-homepage.md`和`spec-daily-news.md`）在 TRAE Work 的 Design模式 选择`21th`设计系统使用以下提示词实现：

```text

用这个设计系统，设计一个每日AI晚报的首页+每日新闻页。

首页：固定顶部导航栏，左侧Logo+文字“每日AI晚报”，右侧历史新闻、关于跳转链接（指向锚点），body内容首先一个卡片内显示三条今日新闻摘要（每条摘要用不同的图标和配色，同每日新闻页）并配一个跳转到新闻详情页面的按钮，然后是历史新闻（时间线+历史新闻摘要+跳转到对应历史新闻详情页面的按钮），最后是关于（关于网站和作者，各一个卡片，并列显示），页脚有年份和“内容由AI生成，请谨慎甄别。”字样。页面右下角要有一键回到顶部的按钮（仅当用户向下滚动页面一段距离后显示）。顶部导航栏和页脚的元素不要贴在屏幕两边，跟body主体宽度一样为宜。

每日新闻页：固定顶部导航栏，左侧Logo+文字“每日AI晚报”，右侧回到首页按钮（完整网址），body内容首先是大字显示标题（即日期），然后每个新闻事件一个卡片，卡片内分事件名称和时间内容，不同类型的新闻要用不同的图标和配色，每条新闻下方显示信息来源的跳转链接。最后添加一个“今日趋势点评”部分，不用卡片。页脚有年份和“内容由AI生成，请谨慎甄别。”字样。页面右下角要有一键回到顶部的按钮（仅当用户向下滚动页面一段距离后显示）。顶部导航栏和页脚的元素不要贴在屏幕两边，跟body主体宽度一样为宜。

注意：两个页面的body内容区宽度要一致。要优化移动端页面布局，尤其是卡片不要在移动端上显示占满整个屏幕宽度，显得拥挤。

最后还有为两个页面分别写一份详尽的设计规范文档。

```
